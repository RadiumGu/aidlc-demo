# AWSomeShop 需求文档

## 意图分析

| 项目 | 内容 |
|------|------|
| **用户需求** | 构建 AWSomeShop 员工福利电商网站 MVP |
| **请求类型** | 新功能实现（在已有骨架代码基础上完成全部 MVP 功能） |
| **范围评估** | 系统级 — 涉及全部 6 个组件（前端 + 5 个后端服务） |
| **复杂度评估** | 中等 — 多服务协同，涉及分布式事务 |

---

## 功能需求

### FR-01: 用户认证（Auth Service）

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-01.1 | 员工使用用户名 + 密码登录系统 | P0 |
| FR-01.2 | 登录成功后签发 JWT Token，返回用户信息和角色 | P0 |
| FR-01.3 | 支持 Token 刷新机制 | P1 |
| FR-01.4 | 密码使用 BCrypt 加密存储 | P0 |
| FR-01.5 | 登录失败次数限制（防暴力破解） | P1 |
| FR-01.6 | 退出登录（Token 失效） | P0 |

**用户模型**:
- 统一用户表，通过角色字段区分（admin / employee）
- 字段：id, username, password, displayName, role, avatar, status, createdAt, updatedAt

### FR-02: 商品管理（Product Service）

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-02.1 | 管理员创建商品（名称、SKU、分类、品牌、积分价格、市场价、库存、描述、图片等） | P0 |
| FR-02.2 | 管理员编辑商品信息 | P0 |
| FR-02.3 | 管理员上下架商品（status 状态管理） | P0 |
| FR-02.4 | 员工浏览商品列表（分页） | P0 |
| FR-02.5 | 员工按分类筛选商品 | P0 |
| FR-02.6 | 员工关键词搜索商品 | P0 |
| FR-02.7 | 员工查看商品详情 | P0 |
| FR-02.8 | 商品库存实时扣减（兑换时） | P0 |

### FR-03: 积分管理（Points Service）

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-03.1 | 员工查看当前积分余额 | P0 |
| FR-03.2 | 员工查看积分变动历史（兑换扣除、管理员发放等） | P0 |
| FR-03.3 | 管理员手动发放积分给指定员工 | P0 |
| FR-03.4 | 管理员手动扣除员工积分 | P0 |
| FR-03.5 | 积分发放/扣除无需审批流程（MVP 阶段） | P0 |
| FR-03.6 | 积分流水记录（操作类型、金额、操作人、时间、备注） | P0 |

**积分模型**:
- 积分账户表：employeeId, balance, totalEarned, totalSpent, updatedAt
- 积分流水表：id, employeeId, type(EARN/SPEND/ADMIN_ADD/ADMIN_DEDUCT), amount, balance, orderId(nullable), operatorId, remark, createdAt

### FR-04: 订单管理（Order Service）

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-04.1 | 员工使用积分兑换商品，创建订单 | P0 |
| FR-04.2 | 订单创建时同步扣减积分和库存（强一致性） | P0 |
| FR-04.3 | 员工查看自己的订单列表 | P0 |
| FR-04.4 | 员工查看订单详情 | P0 |
| FR-04.5 | 管理员查看所有订单 | P0 |
| FR-04.6 | 管理员更新订单状态 | P0 |
| FR-04.7 | 订单取消时退还积分和恢复库存 | P1 |

**订单状态流转**:
```
待处理(PENDING) → 待发货(CONFIRMED) → 已发货(SHIPPED) → 已完成(COMPLETED)
     |                   |
     v                   v
  已取消(CANCELLED)   已取消(CANCELLED)
```

**订单模型**:
- 订单表：id, orderNo, employeeId, totalPoints, status, remark, createdAt, updatedAt
- 订单明细表：id, orderId, productId, productName, pointsPrice, quantity, subtotalPoints

## 需求总结

**核心交付物**：一个完整可运行的 AWSomeShop MVP，包含：
1. **用户认证**: 用户名密码登录、JWT 签发验证、角色区分
2. **商品管理**: 完整 CRUD + 分类筛选 + 搜索
3. **积分系统**: 余额查询、管理员发放/扣除、积分流水
4. **订单系统**: 积分兑换下单、状态流转（待处理→待发货→已发货→已完成/已取消）
5. **分布式事务**: 兑换操作的积分+库存强一致性
6. **前端对接**: 替换所有 Mock 数据，对接真实后端 API
