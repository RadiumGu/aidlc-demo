# 工作单元定义

## 概述

基于已有微服务架构，每个服务自然形成一个独立工作单元。按依赖顺序排列，前序单元完成后后续单元才可开始集成。

---

## Unit 1: Auth Service（认证服务）

| 属性 | 内容 |
|------|------|
| **类型** | 后端微服务 |
| **优先级** | P0（最高 — 全局基础依赖） |
| **范围** | 用户模型、登录认证、JWT 签发/验证、Token 管理 |
| **工作量估算** | 中等 |

**交付物**:
- UserEntity 领域模型 + 数据库迁移
- 登录 API（签发 JWT Token）
- Token 验证内部接口（增强已有壳）
- 退出登录（Token 黑名单）
- 刷新 Token
- 获取当前用户信息
- 登录失败限制
- 密码 BCrypt 加密

---

## Unit 2: Product Service（商品服务）

| 属性 | 内容 |
|------|------|
| **类型** | 后端微服务 |
| **优先级** | P0 |
| **范围** | 补充管理接口、搜索筛选、内部库存操作 |
| **工作量估算** | 中等 |

**交付物**:
- 商品编辑、上下架 API
- 商品列表增强（分类筛选 + 关键词搜索）
- 商品详情 API
- 内部库存扣减/恢复接口（乐观锁）
- 管理接口迁移到 `/admin/` 路径

---

## Unit 3: Points Service（积分服务）

| 属性 | 内容 |
|------|------|
| **类型** | 后端微服务 |
| **优先级** | P0 |
| **范围** | 积分账户、流水、发放/扣除、内部兑换接口 |
| **工作量估算** | 中等 |

**交付物**:
- PointsAccountEntity + PointsTransactionEntity 领域模型 + 数据库迁移
- 积分余额查询 API
- 积分流水查询 API（分页）
- 管理员发放/扣除积分 API
- 内部积分扣减/退还接口（兑换/取消用）

---

## Unit 4: Order Service（订单服务）

| 属性 | 内容 |
|------|------|
| **类型** | 后端微服务 |
| **优先级** | P0 |
| **范围** | 订单创建（Saga 事务）、查询、状态流转、取消 |
| **工作量估算** | 高（涉及跨服务事务编排） |

**交付物**:
- OrderEntity + OrderItemEntity 领域模型 + 数据库迁移
- 兑换下单 API（Saga 编排：扣积分→扣库存→创建订单）
- 我的订单列表 API
- 订单详情 API
- 订单取消 API（退积分+恢复库存）
- 管理员订单列表 + 状态更新 API
- 跨服务 HTTP 客户端（调用 Points/Product 内部接口）

---

## Unit 5: Gateway Service（网关服务）

| 属性 | 内容 |
|------|------|
| **类型** | 后端微服务 |
| **优先级** | P0 |
| **范围** | 新增路由规则、角色权限过滤器 |
| **工作量估算** | 低 |

**交付物**:
- RoleAuthorizationFilter（`/admin/**` 路径需 admin 角色）
- Auth/Points/Order 新路由规则
- AuthValidateResponse 增加 role 字段处理
- X-Operator-Role 头注入

---

## Unit 6: Frontend（前端）

| 属性 | 内容 |
|------|------|
| **类型** | 前端 SPA |
| **优先级** | P0 |
| **范围** | 替换 Mock 数据，对接全部后端 API |
| **工作量估算** | 高（页面多，交互复杂） |

**交付物**:
- 登录对接真实 Auth API
- 员工端：商品列表（筛选+搜索）、商品详情、兑换下单
- 员工端：积分余额、积分流水、我的订单
- 管理端：商品 CRUD、积分发放/扣除、订单管理
- API 服务层封装（各模块 API 函数）
