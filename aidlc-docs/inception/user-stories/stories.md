# 用户故事

组织方式：按角色分组，组内按用户旅程排序。每个故事包含 4-6 条验收标准（含异常场景）。

---

# 🧑‍💼 员工故事集

## Epic 1: 用户认证

### US-01: 员工登录
**作为** 员工李明，
**我想要** 使用用户名和密码登录系统，
**以便** 访问商城并使用积分兑换商品。

**验收标准**:
1. ✅ 输入正确用户名和密码后，成功登录并跳转到商城首页
2. ✅ 登录成功后返回 JWT Token，前端自动存储并在后续请求中携带
3. ✅ 登录成功后页面显示用户名和积分余额
4. ❌ 输入错误密码时，显示"用户名或密码错误"提示
5. ❌ 连续多次登录失败后，账号临时锁定并提示

**关联需求**: FR-01.1, FR-01.2, FR-01.4, FR-01.5

---

### US-02: 员工退出登录
**作为** 员工李明，
**我想要** 退出登录，
**以便** 安全地结束会话。

**验收标准**:
1. ✅ 点击退出后清除本地 Token，跳转到登录页
2. ✅ 退出后使用旧 Token 访问 API 返回 401
3. ✅ 退出后浏览器后退按钮不能回到已认证页面
4. ❌ 网络异常时仍能清除本地状态并跳转登录页

**关联需求**: FR-01.6

---

## Epic 2: 商品浏览

### US-03: 浏览商品列表
**作为** 员工李明，
**我想要** 浏览可用的福利商品列表，
**以便** 找到想要兑换的商品。

**验收标准**:
1. ✅ 商城首页展示商品卡片列表，每张卡片显示图片、名称、积分价格
2. ✅ 支持分页浏览，每页展示合理数量的商品
3. ✅ 仅展示上架状态（status=上架）的商品
4. ✅ 库存为 0 的商品显示"已售罄"标识
5. ❌ 商品列表为空时显示"暂无商品"提示

**关联需求**: FR-02.4, FR-02.8

---

### US-04: 按分类筛选商品
**作为** 员工李明，
**我想要** 按分类筛选商品，
**以便** 快速找到特定类型的商品。

**验收标准**:
1. ✅ 页面展示商品分类标签/下拉框
2. ✅ 选择分类后，列表仅展示该分类下的商品
3. ✅ 支持"全部"选项，清除分类筛选
4. ✅ 筛选结果为空时显示"该分类暂无商品"
5. ✅ 分类筛选可与搜索组合使用

**关联需求**: FR-02.5

---

### US-05: 搜索商品
**作为** 员工李明，
**我想要** 通过关键词搜索商品，
**以便** 快速定位特定商品。

**验收标准**:
1. ✅ 页面提供搜索输入框
2. ✅ 输入关键词后搜索商品名称、描述等字段
3. ✅ 搜索结果实时展示，支持分页
4. ✅ 搜索可与分类筛选组合
5. ❌ 搜索无结果时显示"未找到相关商品"

**关联需求**: FR-02.6

---

### US-06: 查看商品详情
**作为** 员工李明，
**我想要** 查看商品的详细信息，
**以便** 了解商品规格后决定是否兑换。

**验收标准**:
1. ✅ 点击商品卡片进入详情页
2. ✅ 展示商品名称、图片、积分价格、市场价、库存、描述、规格等完整信息
3. ✅ 显示"立即兑换"按钮（积分充足且有库存时可点击）
4. ❌ 积分不足时按钮置灰，显示"积分不足"
5. ❌ 库存不足时按钮置灰，显示"已售罄"

**关联需求**: FR-02.7

---

## Epic 3: 积分兑换

### US-07: 积分兑换下单
**作为** 员工李明，
**我想要** 使用积分兑换商品，
**以便** 享受公司福利。

**验收标准**:
1. ✅ 点击"立即兑换"后弹出确认对话框，显示商品名称和所需积分
2. ✅ 确认后创建订单，同步扣减积分和库存
3. ✅ 兑换成功后显示订单号，跳转到订单详情页
4. ❌ 积分不足时提示"积分余额不足，当前余额 XX"
5. ❌ 库存不足时提示"商品库存不足"
6. ❌ 并发兑换时保证积分和库存一致性，不超卖

**关联需求**: FR-04.1, FR-04.2, NFR-03.1

### US-08: 查看积分余额
**作为** 员工李明，
**我想要** 随时查看我的积分余额，
**以便** 了解可用积分。

**验收标准**:
1. ✅ 页面顶部/侧边栏始终显示当前积分余额
2. ✅ 余额数据实时从后端获取
3. ✅ 兑换成功后余额自动刷新
4. ❌ 接口异常时显示上次缓存的余额并提示"数据可能不是最新"

**关联需求**: FR-03.1

---

### US-09: 查看积分变动历史
**作为** 员工李明，
**我想要** 查看积分变动记录，
**以便** 了解积分的来源和去向。

**验收标准**:
1. ✅ 展示积分流水列表：时间、类型（发放/扣除/兑换）、金额、余额、备注
2. ✅ 按时间倒序排列，支持分页
3. ✅ 兑换类型的流水关联订单号，可点击跳转
4. ❌ 无积分记录时显示"暂无积分变动"

**关联需求**: FR-03.2, FR-03.6

---

### US-10: 查看我的订单
**作为** 员工李明，
**我想要** 查看我的兑换订单列表和详情，
**以便** 跟踪兑换进度。

**验收标准**:
1. ✅ 展示我的订单列表：订单号、商品名、积分数、状态、时间
2. ✅ 支持分页浏览
3. ✅ 点击订单查看详情（商品信息、状态流转时间线）

# 👩‍💼 管理员故事集

### US-11: 管理员登录
**作为** 管理员王芳，
**我想要** 登录管理后台，
**以便** 管理商品、积分和订单。

**验收标准**:

## Epic 6: 商品管理

### US-12: 创建商品
**作为** 管理员王芳，
**我想要** 创建新的福利商品，
**以便** 员工可以浏览和兑换。

**验收标准**:
1. ✅ 填写商品信息表单（名称、SKU、分类、品牌、积分价格、市场价、库存、描述、图片URL等）
2. ✅ 必填字段校验（名称、积分价格、库存）
3. ✅ 创建成功后跳转到商品列表并显示成功提示
4. ❌ SKU 重复时提示"SKU已存在"
5. ❌ 积分价格或库存为负数时表单校验失败

**关联需求**: FR-02.1

---

### US-13: 编辑商品
**作为** 管理员王芳，
**我想要** 编辑已有商品信息，
**以便** 更新价格、库存等。

**验收标准**:
1. ✅ 商品列表中点击"编辑"按钮，进入编辑表单并回填当前数据
2. ✅ 修改后保存成功，列表数据刷新
3. ✅ 支持修改所有商品字段
4. ❌ 编辑不存在的商品返回 404

**关联需求**: FR-02.2

---

### US-14: 商品上下架
**作为** 管理员王芳，
**我想要** 上架或下架商品，
**以便** 控制员工可见的商品范围。

**验收标准**:
1. ✅ 商品列表提供上架/下架切换操作
2. ✅ 下架后员工端不再展示该商品
3. ✅ 上架后员工端可正常浏览和兑换
4. ❌ 有未完成订单的商品下架时给出提示

**关联需求**: FR-02.3

---

## Epic 7: 积分管理

### US-15: 发放积分
**作为** 管理员王芳，
**我想要** 给指定员工发放积分，
**以便** 作为员工福利发放。

**验收标准**:
1. ✅ 选择目标员工、输入积分数量和备注
2. ✅ 发放成功后员工积分余额增加
3. ✅ 生成积分流水记录（类型=管理员发放，含操作人和备注）
4. ✅ 无需审批，直接生效（MVP 阶段）
5. ❌ 积分数量为 0 或负数时校验失败

**关联需求**: FR-03.3, FR-03.5, FR-03.6

---

### US-16: 扣除积分
**作为** 管理员王芳，
**我想要** 扣除指定员工的积分，
**以便** 处理积分异常或纠正。

**验收标准**:
1. ✅ 选择目标员工、输入扣除数量和备注
2. ✅ 扣除成功后员工积分余额减少
3. ✅ 生成积分流水记录（类型=管理员扣除）
4. ❌ 扣除后余额变为负数时提示"余额不足，当前余额 XX"
5. ❌ 扣除数量为 0 或负数时校验失败

**关联需求**: FR-03.4, FR-03.6

---

## Epic 8: 订单管理

### US-17: 查看所有订单
**作为** 管理员王芳，
**我想要** 查看所有兑换订单，
**以便** 了解兑换情况并处理发货。

**验收标准**:
1. ✅ 展示所有订单列表：订单号、员工名、商品、积分数、状态、时间
2. ✅ 支持按状态筛选（待处理/待发货/已发货/已完成/已取消）
3. ✅ 支持分页浏览
4. ✅ 点击订单查看详情

**关联需求**: FR-04.5

---

### US-18: 更新订单状态
**作为** 管理员王芳，
**我想要** 更新订单状态（确认发货、标记完成等），
**以便** 推进兑换流程。

**验收标准**:
1. ✅ 待处理订单可操作：确认（→待发货）或取消
2. ✅ 待发货订单可操作：发货（→已发货）或取消
3. ✅ 已发货订单可操作：完成（→已完成）
4. ✅ 状态变更后列表实时刷新
5. ❌ 已完成/已取消的订单不可再变更状态
6. ❌ 取消订单时退还积分和恢复库存

**关联需求**: FR-04.6, FR-04.7
